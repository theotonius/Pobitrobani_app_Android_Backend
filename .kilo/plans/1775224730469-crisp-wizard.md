## বিস্তারিত বাস্তবায়ন পরিকল্পনা (অফলাইন মোড)

### ১. ক্যাশ স্ট্র্যাটেজি উন্নতি (Phase 1 - প্রথমে)

**লক্ষ্য:** API কল ও AI রেসপন্স ক্যাশ করে অফলাইনে ব্যবহারযোগ্য করা

**বাস্তবায়ন:**
```
src/
├── lib/
│   └── cacheManager.ts      # ক্যাশ ম্যানেজমেন্ট
│   └── offlineStorage.ts   # IndexedDB স্টোরেজ
├── hooks/
│   └── useOfflineCache.ts   # ক্যাশ হুক
```

**ক্যাশ স্ট্র্যাটেজি:**
- API রেসপন্স: ২৪ ঘণ্টা ক্যাশ
- AI রেসপন্স: ৭ দিন ক্যাশ
- ব্যবহার: Cache-first strategy

**IndexedDB স্ট্রাকচার:**
- `verse_cache`: পদ ক্যাশ
- `ai_response_cache`: AI রেসপন্স ক্যাশ
- `user_data`: বুকমার্ক, স্নিপেট

### ২. সংরক্ষিত ডেটা অফলাইন (Phase 2)

**বর্তমান স্টোরেজ:** localStorage (সীমিত)

**উন্নতি:** IndexedDB ব্যবহার
- সংরক্ষিত পদ (unlimited)
- স্নিপেট (unlimited)
- সার্চ ইতিহাস
- সেটিংস

### ৩. অফলাইন ইন্ডিকেটর UI উন্নতি (Phase 3)

**বর্তমান:** শুধু টেক্সট ইন্ডিকেটর

**উন্নতি:**
- অ্যানিমেটেড অফলাইন ব্যানার
- ক্যাশ স্ট্যাটাস ইন্ডিকেটর
- "ক্যাশ থেকে দেখা হচ্ছে" মেসেজ

### ৪. স্থানীয় বাইবেল ডেটা (Phase 4 - শেষে)

**উপায়:**
- JSON ফাইল হিসেবে বাইবেল টেক্সট বান্ডেল
- প্রয়োজনমতো লোড (Lazy load by book)
- প্রাথমিক: ৩টি বাংলা ভার্শনের টেক্সট

**ফাইল স্ট্রাকচার:**
```
public/
└── data/
    ├── bible-modern.json    # ~3MB
    ├── bible-carey.json    # ~3MB
    └── bible-kitabul.json  # ~3MB
```
